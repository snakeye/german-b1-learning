# Error Log

## Prinzip

Hier werden nur echte Fehler oder wiederkehrende Schwächen gesammelt.
Keine abstrakte Theorie.

## Felder pro Fehler

- Datum
- Bereich
- Problem
- Status (`open`, `drilling`, `improving`, `stable`)
- Link zur Detailnotiz
